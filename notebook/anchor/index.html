Scalable Vector Graphics

I usually don’t pay much attention to Scalable Vector Graphics (SVG). Most of the time, I just copy them from somewhere else and paste them into my code to get the job done. But I’ve always wondered why something filled with symbols and numbers feels so hard to understand.

After spending some time learning the basics, though, SVG actually became pretty fun. I realized I could create some really cool effects with it. The mail icon below, for example, pops open and slides the paper out on hover. Effects like these can be created using only SVG, or combined with CSS if needed.

Elements

When I first got into SVG, it all felt a little confusing. But after learning a few core concepts, it started making much more sense. Creating icons, illustrations, and even simple animations became a lot easier than I expected.

SVG includes many elements for everything from simple shapes to complex diagrams and animations. I’ll go through some of the basics here, and for anything deeper, I usually check the MDN SVG element reference .

svg

The root element of an SVG graphic. It defines the drawing area and contains all shapes, paths, text, and groups.

rect

Used for rectangles and squares. You can control its position, width, height, corner radius, fill, and stroke.

line

Represents a straight line between two points. It becomes visible when a stroke is applied.

circle

Draws a circle using its center position and radius.

ellipse

Defines an oval shape. Unlike a circle, it uses separate horizontal and vertical radii.

polyline

Connects multiple points with a series of lines without closing the shape.

polygon

Forms a closed shape from multiple connected points.

path

The most powerful SVG shape. It can draw lines, curves, arcs, and complex custom illustrations using path commands.

Attributes

SVG elements use attributes to control how they look and behave. Here are some of the attributes I ended up using the most. Some only work with specific elements, so I still check the MDN SVG attribute reference whenever I want to learn more.

viewBox

Defines internal coordinate system and determines how graphics scale within the container.

width, height

Control how large the SVG appears on the page.

fill

Applies colors or gradients to a shape.

stroke

Specifies the outline color of a shape, including gradients.

stroke-width

Determines the thickness of a shape or text outline.

stroke-dasharray

Creates dash and gap patterns along an outline.

stroke-linecap

Controls the appearance of the ends of stroke.

Now that I’ve gone through the basics, I want to show how I created a simple menu icon.

Create an <svg> element with a viewBox of 0 0 24 24, defining a 24x24 drawing area starting at (0, 0).

Draw a <line> from (5, 12) to (19, 12), then set its stroke color and width.

Duplicate two more <line>s with different y values.

Add a element at (1, 1), width and height of 22 for the border.

Group shared attributes with <g>, apply stroke-linecap="round", and set redius of 4 to <rect>.
